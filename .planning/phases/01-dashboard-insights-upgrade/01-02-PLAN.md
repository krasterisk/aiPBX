  <action>
Add private method `generateInsights(ctx: InsightsGenerationContext)` that:
1. Reads env: `OPERATOR_INSIGHTS_MIN_CALLS` (default 10), `OPERATOR_INSIGHTS_TTL_SEC` (3600), `OPERATOR_INSIGHTS_MAX_COUNT` (6)
2. Calls existing `getDashboard` (or equivalent) to load full dashboard payload
3. Short-circuits if `sampleSize &lt; MIN_CALLS` returning `{ insights: [], generatedAt, promptVersion, sampleSize, lowConfidence: true }` without LLM (unless cached full response exists)
4. Builds facts via `buildInsightsFacts`, prompt via `buildInsightsPrompt`
5. Computes cache key: `insights:v1:{tenantUserId}:{projectId}:{start}:{end}:{operator}:{promptVersion}:{factsDigest}` where `tenantUserId = ctx.query.userId ?? ctx.realUserId ?? 'admin-all'` and `factsDigest` = first 8 chars of sha256(JSON.stringify(facts))
6. Skips cache when `ctx.query.refresh === '1'` or `ctx.query.refresh === true`
7. Calls `chatWithFallback` with `response_format: { type: 'json_schema', json_schema: buildInsightsJsonSchema() }`, `temperature: 0`
8. Parses via `parseAndValidateInsightsResponse`, truncates to MAX_COUNT
9. Stores in in-memory Map with TTL
10. Invokes existing billing for type `insight` (preserve current billing hook)

Refactor `getInsights` and `getProjectInsights` to thin wrappers calling `generateInsights`. Remove duplicate inline prompt strings and direct `openAiClient` usage for insights.

Sync `insightsAvailable` computation in dashboard response to use same `OPERATOR_INSIGHTS_MIN_CALLS` threshold.
